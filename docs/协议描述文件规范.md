# DeviceStudio 协议描述文件规范

> 版本：v1.0
> 状态：设计阶段

---

## 一、概述

协议描述文件用于定义数据帧的解析规则，采用 JSON 格式，使协议解析器能够自动解析设备通信数据。

### 1.1 设计目标

| 目标 | 说明 |
|-----|------|
| **易读性** | JSON 格式，人眼可读 |
| **完整性** | 覆盖常见协议类型 |
| **可扩展** | 支持自定义字段类型 |
| **跨平台** | 无平台相关内容 |

### 1.2 文件位置

```
config/protocols/
├── default.json           # 默认协议
├── modbus_rtu.json        # Modbus RTU 协议
├── modbus_tcp.json        # Modbus TCP 协议
├── custom/                # 用户自定义协议
│   ├── my_protocol.json
│   └── sensor_v2.json
└── templates/             # 协议模板
    ├── basic_template.json
    └── advanced_template.json
```

## 二、文件结构

### 2.1 顶层结构

```json
{
    "meta": { ... },           // 元数据（必填）
    "frame_definitions": [ ... ], // 帧定义列表（必填）
    "enums": { ... },          // 枚举定义（可选）
    "constants": { ... },      // 常量定义（可选）
    "functions": { ... }       // 自定义函数（可选）
}
```

### 2.2 元数据 (meta)

```json
{
    "meta": {
        "name": "MyProtocol",
        "version": "1.0.0",
        "description": "自定义传感器协议",
        "author": "DeviceStudio",
        "created": "2025-02-14",
        "modified": "2025-02-14"
    }
}
```

| 字段 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| name | string | 是 | 协议名称，用于显示和识别 |
| version | string | 是 | 协议版本，格式：主版本.次版本.修订号 |
| description | string | 否 | 协议描述 |
| author | string | 否 | 作者信息 |
| created | string | 否 | 创建日期 |
| modified | string | 否 | 最后修改日期 |

## 三、帧定义 (frame_definitions)

### 3.1 基本帧结构

```json
{
    "frame_definitions": [
        {
            "name": "SensorData",
            "description": "传感器数据帧",
            "identification": { ... },
            "structure": { ... },
            "fields": [ ... ],
            "validation": { ... }
        }
    ]
}
```

### 3.2 帧识别规则 (identification)

#### 3.2.1 帧头识别

```json
{
    "identification": {
        "header": "AA55",
        "header_mask": "FFFF"  // 可选，用于部分匹配
    }
}
```

#### 3.2.2 长度识别

```json
{
    "identification": {
        "length_field": {
            "offset": 2,
            "size": 2,
            "byte_order": "little_endian",
            "length_type": "payload_only"  // 或 "total_frame"
        }
    }
}
```

| length_type | 说明 |
|-------------|------|
| `payload_only` | 长度字段表示负载长度 |
| `total_frame` | 长度字段表示整个帧长度 |
| `remaining` | 长度字段表示剩余字节数 |

#### 3.2.3 组合识别

```json
{
    "identification": {
        "header": "AA55",
        "length_field": {
            "offset": 2,
            "size": 2,
            "byte_order": "big_endian"
        },
        "min_length": 8,
        "max_length": 1024
    }
}
```

### 3.3 帧结构 (structure)

```json
{
    "structure": {
        "preamble": {
            "offset": 0,
            "size": 2
        },
        "length": {
            "offset": 2,
            "size": 2
        },
        "payload": {
            "offset": 4,
            "size_from": "length"
        },
        "crc": {
            "offset": -2,
            "size": 2,
            "relative": true  // 相对于帧尾
        }
    }
}
```

### 3.4 字段定义 (fields)

#### 3.4.1 基本字段类型

| 类型 | 大小 | 说明 | 示例 |
|-----|------|------|------|
| `uint8` | 1 | 无符号8位整数 | 0-255 |
| `int8` | 1 | 有符号8位整数 | -128~127 |
| `uint16` | 2 | 无符号16位整数 | 0-65535 |
| `int16` | 2 | 有符号16位整数 | -32768~32767 |
| `uint32` | 4 | 无符号32位整数 | 0-4294967295 |
| `int32` | 4 | 有符号32位整数 | -2147483648~2147483647 |
| `uint64` | 8 | 无符号64位整数 | - |
| `int64` | 8 | 有符号64位整数 | - |
| `float32` | 4 | 单精度浮点数 | IEEE 754 |
| `float64` | 8 | 双精度浮点数 | IEEE 754 |
| `string` | 变长 | 字符串 | ASCII/UTF-8 |
| `bytes` | 变长 | 原始字节 | - |
| `bool` | 1 | 布尔值 | 0/1 |
| `bits` | 变长 | 位字段 | 按位解析 |

#### 3.4.2 字段定义示例

```json
{
    "fields": [
        {
            "name": "device_id",
            "offset": 4,
            "type": "uint8",
            "description": "设备ID"
        },
        {
            "name": "temperature",
            "offset": 5,
            "type": "int16",
            "byte_order": "big_endian",
            "scale": 0.1,
            "unit": "°C",
            "description": "温度值"
        },
        {
            "name": "humidity",
            "offset": 7,
            "type": "uint16",
            "byte_order": "big_endian",
            "scale": 0.01,
            "unit": "%",
            "description": "湿度值"
        },
        {
            "name": "status",
            "offset": 9,
            "type": "uint8",
            "description": "状态字节",
            "bits": [
                {"name": "is_running", "bit": 0, "description": "运行状态"},
                {"name": "has_error", "bit": 1, "description": "错误标志"},
                {"name": "is_charging", "bit": 2, "description": "充电状态"},
                {"name": "reserved", "bits": "3-7", "description": "保留"}
            ]
        },
        {
            "name": "timestamp",
            "offset": 10,
            "type": "uint32",
            "byte_order": "little_endian",
            "description": "时间戳",
            "format": "unix_timestamp"
        },
        {
            "name": "device_name",
            "offset": 14,
            "type": "string",
            "length": 16,
            "encoding": "utf-8",
            "description": "设备名称"
        }
    ]
}
```

### 3.5 字段属性详解

| 属性 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| name | string | 是 | 字段名称，使用 snake_case |
| offset | int | 是 | 字段偏移量（字节） |
| type | string | 是 | 数据类型 |
| length | int | 条件 | 字段长度（string/bytes 必填） |
| byte_order | string | 否 | 字节序：`little_endian` / `big_endian`，默认 `big_endian` |
| scale | number | 否 | 缩放因子，最终值 = 原始值 × scale + offset_value |
| offset_value | number | 否 | 偏移值 |
| unit | string | 否 | 单位 |
| description | string | 否 | 描述 |
| encoding | string | 否 | 字符串编码：`ascii` / `utf-8` / `gbk` |
| format | string | 否 | 格式化类型 |
| enum | string | 否 | 枚举类型引用 |
| default | any | 否 | 默认值 |
| min | number | 否 | 最小值 |
| max | number | 否 | 最大值 |
| bits | array | 否 | 位字段定义 |

## 四、校验规则 (validation)

### 4.1 CRC 校验

```json
{
    "validation": {
        "crc": {
            "algorithm": "crc16_modbus",
            "offset": -2,
            "size": 2,
            "data_range": {
                "start": 0,
                "end": -3
            }
        }
    }
}
```

#### 支持的 CRC 算法

| 算法名称 | 多项式 | 初始值 | 说明 |
|---------|--------|--------|------|
| `crc8` | 0x07 | 0x00 | CRC-8 |
| `crc8_ccitt` | 0x07 | 0xFF | CRC-8-CCITT |
| `crc16_modbus` | 0x8005 | 0xFFFF | Modbus CRC-16 |
| `crc16_ccitt` | 0x1021 | 0xFFFF | CRC-16-CCITT |
| `crc16_xmodem` | 0x1021 | 0x0000 | XMODEM CRC-16 |
| `crc32` | 0x04C11DB7 | 0xFFFFFFFF | CRC-32 |
| `crc32c` | 0x1EDC6F41 | 0xFFFFFFFF | CRC-32C |

### 4.2 校验和

```json
{
    "validation": {
        "checksum": {
            "type": "xor",
            "offset": -1,
            "data_range": {
                "start": 0,
                "end": -2
            }
        }
    }
}
```

#### 校验和类型

| 类型 | 说明 |
|-----|------|
| `sum` | 累加和（取低字节） |
| `xor` | 异或校验 |
| `lrc` | 纵向冗余校验 |

### 4.3 固定值校验

```json
{
    "validation": {
        "fixed_value": {
            "offset": 0,
            "value": "AA55",
            "description": "帧头固定值"
        }
    }
}
```

### 4.4 组合校验

```json
{
    "validation": {
        "rules": [
            {
                "type": "header",
                "value": "AA55"
            },
            {
                "type": "crc",
                "algorithm": "crc16_modbus",
                "offset": -2
            },
            {
                "type": "length",
                "min": 8,
                "max": 256
            }
        ]
    }
}
```

## 九、解析器行为

### 9.1 解析流程

```
输入数据流
    │
    ▼
┌─────────────┐
│  帧识别     │ ← 根据帧头/长度规则识别帧边界
└─────────────┘
    │
    ▼
┌─────────────┐
│  字段提取   │ ← 按字段定义提取各字段值
└─────────────┘
    │
    ▼
┌─────────────┐
│  类型转换   │ ← 字节序转换、类型转换
└─────────────┘
    │
    ▼
┌─────────────┐
│  值处理     │ ← 缩放、偏移、枚举映射
└─────────────┘
    │
    ▼
┌─────────────┐
│  校验验证   │ ← CRC/校验和验证
└─────────────┘
    │
    ▼
输出结构化数据
```

### 9.2 错误处理

| 错误类型 | 说明 | 处理方式 |
|---------|------|---------|
| `IncompleteFrame` | 数据不完整 | 等待更多数据 |
| `InvalidHeader` | 帧头不匹配 | 丢弃数据，重新同步 |
| `InvalidLength` | 长度字段错误 | 标记错误，跳过该帧 |
| `CrcError` | CRC 校验失败 | 标记错误，可配置是否保留数据 |
| `UnknownFieldType` | 未知字段类型 | 忽略该字段 |
| `OutOfRange` | 值超出范围 | 标记警告，保留原始值 |

## 十、扩展机制

### 10.1 自定义类型

可通过插件注册自定义字段类型：

```cpp
// C++ 插件注册自定义类型
class CustomTypeHandler : public IFieldTypeHandler {
public:
    QString typeName() const override { return "bcd"; }
    QVariant parse(const QByteArray& data, int offset, const QVariantMap& config) override {
        // BCD 解码逻辑
        quint8 bcd = static_cast<quint8>(data[offset]);
        return (bcd >> 4) * 10 + (bcd & 0x0F);
    }
};
```
