> 版本：v1.0

## 一、概述

本文档描述 DeviceStudio 在 Windows 和 Linux 平台上的安装部署流程。

### 1.1 支持的平台

| 平台 | 版本要求 | 架构 |
|-----|---------|------|
| Windows 10 | 1903+ | x64 |
| Windows 11 | - | x64 |
| Ubuntu | 22.04 LTS | x64 |
| Debian | 12+ | x64 |
| CentOS | 8+ | x64 |

### 1.2 系统要求

| 项目 | 最低配置 | 推荐配置 |
|-----|---------|---------|
| CPU | 2核 | 4核+ |
| 内存 | 4GB | 8GB+ |
| 磁盘 | 500MB | 2GB+ |
| 显卡 | 支持 OpenGL 2.0 | 独立显卡 |

## 三、Linux 安装

### 3.1 安装包格式

| 格式 | 适用发行版 |
|-----|-----------|
| .deb | Debian, Ubuntu |
| .rpm | CentOS, Fedora, openSUSE |
| .AppImage | 通用（免安装） |

### 3.2 Debian/Ubuntu 安装

#### 3.2.1 依赖安装

```bash
# 安装依赖
sudo apt-get update
sudo apt-get install -y \
    libqt6core6 \
    libqt6gui6 \
    libqt6widgets6 \
    libqt6serialport6 \
    libstdc++6
```

#### 3.2.2 安装 .deb 包

```bash
# 下载安装包
wget https://example.com/devicestudio/device-studio_1.0.0_amd64.deb

# 安装
sudo dpkg -i device-studio_1.0.0_amd64.deb

# 如果有依赖问题，修复
sudo apt-get install -f
```

#### 3.2.3 验证安装

```bash
# 检查安装
dpkg -l | grep device-studio

# 运行
device-studio --version
```

### 3.3 CentOS/Fedora 安装

#### 3.3.1 依赖安装

```bash
# CentOS 8
sudo dnf install -y \
    qt6-qtbase \
    qt6-qtbase-gui \
    qt6-qtserialport \
    libstdc++
```

#### 3.3.2 安装 .rpm 包

```bash
# 下载安装包
wget https://example.com/devicestudio/device-studio-1.0.0-1.x86_64.rpm

# 安装
sudo rpm -ivh device-studio-1.0.0-1.x86_64.rpm

# 或使用 dnf
sudo dnf install device-studio-1.0.0-1.x86_64.rpm
```

### 3.4 AppImage 安装（推荐）

#### 3.4.1 下载与安装

```bash
# 下载 AppImage
wget https://example.com/devicestudio/DeviceStudio-1.0.0-x86_64.AppImage

# 添加执行权限
chmod +x DeviceStudio-1.0.0-x86_64.AppImage

# 运行
./DeviceStudio-1.0.0-x86_64.AppImage
```

#### 3.4.2 桌面集成

```bash
# 提取图标和桌面文件
./DeviceStudio-1.0.0-x86_64.AppImage --appimage-extract

# 复制桌面文件
cp squashfs-root/device-studio.desktop ~/.local/share/applications/

# 复制图标
cp squashfs-root/device-studio.png ~/.local/share/icons/

# 更新桌面数据库
update-desktop-database ~/.local/share/applications
```

### 3.5 从源码编译安装

#### 3.5.1 安装编译依赖

**Ubuntu 22.04:**

```bash
sudo apt-get install -y \
    build-essential \
    cmake \
    git \
    qt6-base-dev \
    qt6-serialport-dev \
    qt6-tools-dev \
    ninja-build
```

**CentOS 8:**

```bash
sudo dnf groupinstall -y "Development Tools"
sudo dnf install -y \
    cmake \
    git \
    qt6-qtbase-devel \
    qt6-qtserialport-devel \
    ninja-build
```

#### 3.5.2 编译步骤

```bash
# 克隆仓库
git clone https://github.com/your-org/DeviceStudio.git
cd DeviceStudio

# 创建构建目录
mkdir build && cd build

# 配置 (Release 版本)
cmake .. \
    -DCMAKE_BUILD_TYPE=Release \
    -DCMAKE_INSTALL_PREFIX=/usr/local \
    -DBUILD_TESTS=OFF

# 编译
cmake --build . --parallel $(nproc)

# 安装
sudo cmake --install .
```

#### 3.5.3 创建 DEB/RPM 包

```bash
# 创建 DEB 包
cpack -G DEB

# 创建 RPM 包
cpack -G RPM
```

### 4.2 串口权限配置（Linux）

#### 4.2.1 添加用户到 dialout 组

```bash
# 添加当前用户到 dialout 组
sudo usermod -a -G dialout $USER

# 注销并重新登录生效

# 验证
groups $USER
# 输出应包含 dialout
```

#### 4.2.2 创建 udev 规则（可选）

```bash
# 创建规则文件
sudo nano /etc/udev/rules.d/99-usb-serial.rules

# 添加规则（示例：CP210x 设备）
SUBSYSTEM=="tty", ATTRS{idVendor}=="10c4", ATTRS{idProduct}=="ea60", MODE="0666"

# 重新加载规则
sudo udevadm control --reload-rules
sudo udevadm trigger
```

### 4.3 防火墙配置

#### 4.3.1 Windows 防火墙

```powershell
# 允许程序
netsh advfirewall firewall add rule name="DeviceStudio" dir=in action=allow program="C:\Program Files\DeviceStudio\DeviceStudio.exe" enable=yes

# 允许端口（TCP 服务器模式）
netsh advfirewall firewall add rule name="DeviceStudio-TCP" dir=in action=allow protocol=tcp localport=8080
```

#### 4.3.2 Linux 防火墙

**ufw (Ubuntu):**

```bash
# 允许端口
sudo ufw allow 8080/tcp

# 查看状态
sudo ufw status
```

**firewalld (CentOS):**

```bash
# 允许端口
sudo firewall-cmd --permanent --add-port=8080/tcp
sudo firewall-cmd --reload
```

### 5.1 Windows 升级

```
方法1: 自动更新
  帮助 → 检查更新 → 下载并安装

方法2: 手动升级
  下载新版本安装包 → 运行安装程序 → 覆盖安装
```

### 5.2 Linux 升级

```bash
# DEB 包升级
sudo dpkg -i device-studio_1.1.0_amd64.deb

# RPM 包升级
sudo rpm -Uvh device-studio-1.1.0-1.x86_64.rpm

# AppImage 升级

```

### 5.3 配置迁移

升级时配置文件会自动保留，建议升级前备份：

**Windows:**
```powershell
# 备份
Copy-Item "$env:APPDATA\DeviceStudio" "$env:APPDATA\DeviceStudio_backup" -Recurse
```

**Linux:**
```bash
# 备份
cp -r ~/.config/DeviceStudio ~/.config/DeviceStudio_backup
```

## 六、常见问题

### 6.1 Windows 问题

| 问题 | 原因 | 解决方案 |
|-----|------|---------|
| 无法启动，缺少 DLL | 运行时库缺失 | 安装 Visual C++ Redistributable |
| 串口无法打开 | 驱动未安装 | 安装 USB-Serial 驱动 |
| 程序崩溃 | 兼容性问题 | 右键 → 属性 → 兼容性 → 以兼容模式运行 |
| 中文乱码 | 编码问题 | 设置系统区域为 UTF-8 |

### 6.2 Linux 问题

| 问题 | 原因 | 解决方案 |
|-----|------|---------|
| 权限被拒绝 | 不在 dialout 组 | `sudo usermod -a -G dialout $USER` |
| 无法连接显示 | DISPLAY 变量 | `export DISPLAY=:0` |
| AppImage 不运行 | FUSE 缺失 | `sudo apt install fuse libfuse2` |
| Qt 插件错误 | Qt 环境变量 | 检查 QT_PLUGIN_PATH |

### 6.3 网络问题

| 问题 | 原因 | 解决方案 |
|-----|------|---------|
| TCP 连接失败 | 防火墙阻止 | 开放对应端口 |
| 连接超时 | 网络不通 | 检查网络和 IP 地址 |
| UDP 收不到数据 | 绑定地址错误 | 使用 0.0.0.0 绑定所有接口 |

## 七、企业部署

### 7.1 批量安装脚本

**Windows (PowerShell):**

```powershell
# install_devicestudio.ps1

$installerPath = "\\server\share\DeviceStudio-1.0.0-windows-x64.exe"
$installDir = "C:\Program Files\DeviceStudio"

# 静默安装
Start-Process -FilePath $installerPath -ArgumentList "/S", "/D=$installDir" -Wait

# 导入配置
$configSource = "\\server\share\devicestudio_config"
$configDest = "$env:APPDATA\DeviceStudio"
Copy-Item $configSource $configDest -Recurse -Force

Write-Host "DeviceStudio 安装完成"
```

**Linux (Bash):**

```bash
#!/bin/bash
# install_devicestudio.sh

INSTALLER="device-studio_1.0.0_amd64.deb"
CONFIG_SOURCE="/srv/share/devicestudio_config"

# 安装
dpkg -i $INSTALLER
apt-get install -f -y

# 导入配置
mkdir -p ~/.config/DeviceStudio
cp -r $CONFIG_SOURCE/* ~/.config/DeviceStudio/

echo "DeviceStudio 安装完成"
```
