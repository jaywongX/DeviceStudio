## 一、产品概述

### 1.1 产品定位

**DeviceStudio** 是一款面向嵌入式开发、物联网开发、硬件工程师的**通用设备调试与数据分析平台**。

核心目标：
- 统一调试各种设备
- 可视化设备数据
- 支持协议解析
- 支持自动化测试
- 支持插件扩展

### 1.2 产品愿景

打造一款**工业级调试平台**，类似软件包括：
- 串口调试助手
- Modbus Poll
- Hterm
- Wireshark（部分理念）

但 DeviceStudio 更注重**模块化**、**可扩展**、**智能化**。

## 二、目标用户分析

### 2.1 嵌入式工程师

| 需求类别 | 具体需求 |
|---------|---------|
| 基础需求 | 调试串口、看设备返回数据、快速发送指令 |
| 进阶需求 | 协议解析、自动化测试脚本 |
| 痛点 | 手动数据分析繁琐、协议格式不统一 |

### 2.2 硬件工程师

| 需求类别 | 具体需求 |
|---------|---------|
| 基础需求 | 测试设备通信、查看实时数据 |
| 进阶需求 | 数据曲线分析、仪表盘展示 |
| 痛点 | 数据可视化能力弱、无法批量测试 |

### 2.3 测试工程师

| 需求类别 | 具体需求 |
|---------|---------|
| 基础需求 | 自动化测试、批量设备测试、日志记录 |
| 进阶需求 | 测试脚本编写、测试报告生成 |
| 痛点 | 缺乏通用测试框架、报告不直观 |

### 4.3 协议解析系统（核心亮点）

#### 4.3.1 功能描述

将原始二进制数据解析为结构化数据，便于理解和分析。

#### 4.3.2 解析流程

```
原始数据 (二进制)
      ↓
   帧头识别
      ↓
   字段解析
      ↓
   数据转换
      ↓
   格式化输出
```

#### 4.3.3 协议描述文件格式

使用 JSON 定义协议：

```json
{
  "protocol_name": "温湿度传感器协议",
  "protocol_version": "1.0",
  "frame_header": "AA",
  "frame_tail": "FF",
  "checksum": {
    "enabled": true,
    "type": "xor",
    "range": "data"
  },
  "fields": [
    {
      "name": "header",
      "offset": 0,
      "length": 1,
      "type": "uint8",
      "display_name": "帧头",
      "fixed_value": "AA"
    },
    {
      "name": "device_id",
      "offset": 1,
      "length": 1,
      "type": "uint8",
      "display_name": "设备ID"
    },
    {
      "name": "cmd",
      "offset": 2,
      "length": 1,
      "type": "uint8",
      "display_name": "命令码"
    },
    {
      "name": "temperature",
      "offset": 3,
      "length": 2,
      "type": "int16",
      "scale": 0.1,
      "unit": "℃",
      "display_name": "温度"
    },
    {
      "name": "humidity",
      "offset": 5,
      "length": 2,
      "type": "int16",
      "scale": 0.1,
      "unit": "%RH",
      "display_name": "湿度"
    },
    {
      "name": "checksum",
      "offset": 7,
      "length": 1,
      "type": "uint8",
      "display_name": "校验和"
    },
    {
      "name": "tail",
      "offset": 8,
      "length": 1,
      "type": "uint8",
      "display_name": "帧尾",
      "fixed_value": "FF"
    }
  ]
}
```

#### 4.3.4 解析结果示例

原始数据：`AA 01 03 00 9C 01 F7 8A FF`

解析后：

| 字段名 | 值 | 说明 |
|-------|-----|------|
| 帧头 | 0xAA | 固定 |
| 设备ID | 1 | |
| 命令码 | 3 | |
| 温度 | 25.2℃ | 0x009C = 156 × 0.1 |
| 湿度 | 50.3%RH | 0x01F7 = 503 × 0.1 |
| 校验和 | 0x8A | XOR校验 |
| 帧尾 | 0xFF | 固定 |

#### 4.3.5 支持的数据类型

| 类型 | 说明 | 示例 |
|-----|------|------|
| uint8 | 无符号8位 | 0-255 |
| int8 | 有符号8位 | -128-127 |
| uint16 | 无符号16位 | 0-65535 |
| int16 | 有符号16位 | -32768-32767 |
| uint32 | 无符号32位 | 0-4294967295 |
| int32 | 有符号32位 | -2147483648-2147483647 |

#### 4.3.6 校验和类型

| 类型 | 说明 |
|-----|------|
| xor | 异或校验 |

### 4.5 自动化脚本系统

#### 4.5.1 功能描述

支持自动化测试脚本，提高测试效率。

#### 4.5.2 脚本命令

| 命令 | 说明 | 示例 |
|-----|------|------|
| CONNECT | 连接设备 | `CONNECT COM3 115200` |
| DISCONNECT | 断开连接 | `DISCONNECT` |
| SEND | 发送数据 | `SEND AA010100FF` |
| SENDHEX | 发送HEX | `SENDHEX AA 01 01 00 FF` |
| WAIT | 等待毫秒 | `WAIT 1000` |
| EXPECT | 期望接收 | `EXPECT AA 01` |
| EXPECTHEX | HEX期望 | `EXPECTHEX AA0101` |
| IF/ELSE/ENDIF | 条件判断 | `IF $var == 1 ... ENDIF` |
| FOR/ENDFOR | 循环 | `FOR i=1 TO 10 ... ENDFOR` |
| SET | 变量赋值 | `SET $temp = 25` |
| LOG | 记录日志 | `LOG 测试开始` |
| DELAY | 延迟 | `DELAY 100` |

#### 4.5.3 脚本示例

**示例1：基础通信测试**

```bash
# 连接串口
CONNECT COM3 115200

# 发送查询命令
SEND AA010100FF

# 等待响应
WAIT 500

# 检查响应
EXPECTHEX AA 01 02

# 记录结果
LOG 测试通过

# 断开连接
DISCONNECT
```

**示例2：循环批量测试**

```bash
# 连接设备
CONNECT TCP 192.168.1.100 5000

# 循环测试10次
FOR i=1 TO 10
    LOG 第 $i 次测试
    
    # 发送数据
    SEND AA01
    
    # 等待响应
    WAIT 200
    
    # 检查是否成功
    IF $RESPONSE == "OK"
        LOG 测试成功
    ELSE
        LOG 测试失败
        BREAK
    ENDIF
ENDFOR

DISCONNECT
```

**示例3：数据采集脚本**

```bash
CONNECT COM4 9600

# 设置采集参数
SET $SAMPLES = 100
SET $INTERVAL = 100

# 开始采集
FOR $i = 1 TO $SAMPLES
    SEND QUERY
    
    WAIT $INTERVAL
    
    # 提取温度数据
    EXTRACT temperature FROM RESPONSE
    
    # 记录到日志
    LOG 采样 $i: $temperature
    
    # 发送到图表
    PLOT temperature $temperature
ENDFOR

LOG 采集完成
DISCONNECT
```

#### 4.7.2 插件接口定义

**协议插件接口：**

```cpp
class IProtocolPlugin
{
public:
    virtual ~IProtocolPlugin() = default;
    
    // 插件名称
    virtual QString name() const = 0;
    
    // 插件版本
    virtual QString version() const = 0;
    
    // 协议描述
```

使用 **CMake 3.16+** 作为跨平台构建系统。

#### 7.2.1 根目录 CMakeLists.txt 示例

```cmake
cmake_minimum_required(VERSION 3.16)

project(DeviceStudio VERSION 1.0.0 LANGUAGES CXX)

# C++17 标准
set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)
set(CMAKE_CXX_EXTENSIONS OFF)

# 自动处理 Qt MOC, UIC, RCC
set(CMAKE_AUTOMOC ON)
set(CMAKE_AUTOUIC ON)
set(CMAKE_AUTORCC ON)

# 查找 Qt6 (优先) 或 Qt5
find_package(QT NAMES Qt6 Qt5 REQUIRED COMPONENTS Core Widgets SerialPort Network)
find_package(Qt${QT_VERSION_MAJOR} REQUIRED COMPONENTS Core Widgets SerialPort Network)

# 平台特定设置
if(WIN32)
    # Windows MSVC 设置
    if(MSVC)
        # 设置 MSVC 运行时库
        set(CMAKE_MSVC_RUNTIME_LIBRARY "MultiThreaded$<$<CONFIG:Debug>:Debug>DLL")
        
        # MSVC 编译选项
        add_compile_options(
            /W4           # 警告级别4
            /utf-8        # UTF-8 源码
            /MP           # 多进程编译
        )
        
        # 定义 Windows 版本
        add_definitions(-DWIN32_LEAN_AND_MEAN)
        add_definitions(-DNOMINMAX)
    endif()
    
elseif(UNIX AND NOT APPLE)
    # Linux GCC 设置
    add_compile_options(
        -Wall
        -Wextra
        -Wpedantic
        -Werror=return-type
    )
    
    # 链接线程库
    find_package(Threads REQUIRED)
    
    # Linux 特定定义
    add_definitions(-DLINUX)
endif()

# 包含目录
include_directories(${CMAKE_SOURCE_DIR}/src)

# 添加子目录
add_subdirectory(src/core)
add_subdirectory(src/communication)
add_subdirectory(src/visualization)
add_subdirectory(src/ui)
add_subdirectory(src/plugin)
add_subdirectory(src/utils)

# 可执行文件
add_executable(${PROJECT_NAME}
    src/main/main.cpp
)

# 链接库
target_link_libraries(${PROJECT_NAME} PRIVATE
    core
    communication
    visualization
    ui
    plugin
    utils
    Qt${QT_VERSION_MAJOR}::Core
    Qt${QT_VERSION_MAJOR}::Widgets
    Qt${QT_VERSION_MAJOR}::SerialPort
    Qt${QT_VERSION_MAJOR}::Network
)

# Linux 链接线程库
if(UNIX AND NOT APPLE)
    target_link_libraries(${PROJECT_NAME} PRIVATE Threads::Threads)
endif()

# 安装规则
install(TARGETS ${PROJECT_NAME}
    RUNTIME DESTINATION bin
    BUNDLE DESTINATION .
)

# 复制资源文件
install(DIRECTORY resources/ DESTINATION resources)
install(DIRECTORY config/ DESTINATION config)
```
