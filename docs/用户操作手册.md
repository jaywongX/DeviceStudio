## 一、产品简介

### 1.1 产品概述

**DeviceStudio** 是一款面向嵌入式开发、物联网开发、硬件工程师的**通用设备调试与数据分析平台**。

### 1.2 主要功能

| 功能模块 | 说明 |
|---------|------|
| 设备管理 | 管理串口、TCP、UDP 等多种设备连接 |
| 数据收发 | 数据发送、接收、显示和记录 |
| 协议解析 | 自动解析数据帧，显示结构化数据 |
| 实时可视化 | 曲线图表、仪表盘、监控面板 |
| 自动化脚本 | 支持脚本自动化测试和控制 |
| 数据日志 | 通信日志记录、搜索和导出 |
| 插件扩展 | 支持自定义协议和功能扩展 |

## 三、设备管理

### 3.1 设备类型

| 类型 | 说明 | 典型用途 |
|-----|------|---------|
| 串口设备 | RS232/RS485 串口通信 | 传感器、调试器 |
| TCP 客户端 | 作为客户端连接服务器 | PLC、IoT 设备 |

| Modbus RTU | Modbus 串口协议 | 工业设备 |
| Modbus TCP | Modbus TCP 协议 | 工业以太网 |

## 四、协议解析

### 4.1 协议文件格式

协议文件使用 JSON 格式，定义数据帧的解析规则。

```json
{
    "meta": {
        "name": "TemperatureSensor",
        "version": "1.0.0"
    },
    "frame_definitions": [
        {
            "name": "SensorData",
            "identification": {
                "header": "AA55",
                "min_length": 6
            },
            "fields": [
                {"name": "device_id", "offset": 2, "type": "uint8"},
                {"name": "temperature", "offset": 3, "type": "int16", "scale": 0.1},
                {"name": "checksum", "offset": 5, "type": "uint8"}
            ]
        }
    ]
}
```

## 六、自动化脚本

### 6.1 脚本语言

DeviceStudio 使用简化的脚本语言进行自动化控制。

#### 6.1.1 基本语法

```
# 注释以 # 开头

# 设备操作
CONNECT <设备名>              # 连接设备
DISCONNECT <设备名>           # 断开设备

# 数据发送
SEND <数据>                   # 发送 HEX 数据
SEND <设备> <数据>            # 发送到指定设备
SEND_ASCII <字符串>           # 发送 ASCII 字符串

# 等待和延时
WAIT <毫秒>                   # 等待指定时间
WAIT_DATA <超时>              # 等待数据到达

# 条件判断
EXPECT <模式> [超时]          # 期望接收指定数据
IF <条件> THEN ... END        # 条件执行

# 循环
FOR i=<开始>..<结束>          # 循环
  ...
END

# 变量
SET <变量名> <值>             # 设置变量
PRINT <消息>                  # 打印消息
```

#### 6.1.2 脚本示例

**示例 1：简单发送接收**

```
# 连接设备
CONNECT COM3

# 发送查询命令
SEND AA 55 01 00

# 等待响应
WAIT 1000

# 期望收到确认
EXPECT AA 55 01 01

# 断开连接
DISCONNECT COM3
```

**示例 2：循环测试**

```
# 初始化
SET count 0
SET pass 0
SET fail 0

# 连接设备
CONNECT COM3

# 循环测试 10 次
FOR i=1..10
    # 发送测试命令
    SEND AA 55 01 $i
    
    # 等待响应
    WAIT 500
    
    # 检查响应
    IF EXPECT AA 55 01 OK 1000 THEN
        SET pass pass + 1
        PRINT "测试 $i 通过"
    ELSE
        SET fail fail + 1
        PRINT "测试 $i 失败"
    END
    
    SET count count + 1
END

# 打印统计
PRINT "测试完成: 通过 $pass, 失败 $fail"

# 断开连接
DISCONNECT COM3
```

## 九、常见问题

### 9.1 连接问题

**Q: 串口无法打开？**

A: 检查以下几点：
1. 串口是否被其他程序占用
2. 串口权限是否正确（Linux 需要在 dialout 组）
3. 串口参数是否与设备匹配
4. USB 转串口驱动是否安装

**Q: TCP 连接超时？**

A: 检查以下几点：
1. 目标 IP 地址是否正确
2. 目标端口是否开放
3. 防火墙是否阻止连接
4. 网络是否连通（ping 测试）

### 9.2 数据问题

**Q: 接收数据乱码？**

A: 检查以下几点：
1. 波特率是否匹配
2. 数据位/校验位/停止位是否正确
3. 显示格式是否正确（HEX/ASCII）

**Q: 数据解析失败？**

A: 检查以下几点：
1. 协议文件是否正确加载
2. 帧定义是否与实际数据匹配
3. 帧头/长度/校验是否正确

### 9.3 性能问题

**Q: 程序运行缓慢？**

A: 尝试以下优化：
1. 减少可视化窗口数量
2. 降低数据采样率
3. 减少日志级别
4. 清理历史数据
