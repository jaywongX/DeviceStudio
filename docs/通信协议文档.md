## 一、概述

本文档定义 DeviceStudio 支持的通信协议类型、参数配置和协议格式。

### 1.1 支持的通信方式

| 通信方式 | 协议类型 | 典型应用场景 |
|---------|---------|-------------|
| 串口通信 | Serial | 嵌入式设备调试、传感器数据采集 |
| TCP 客户端 | TCP Client | 网络设备通信、IoT 设备 |
| TCP 服务器 | TCP Server | 多客户端数据接收 |
| UDP 通信 | UDP | 广播/组播数据接收 |
| Modbus RTU | Serial | 工业设备通信 |
| Modbus TCP | TCP | 工业以太网设备 |
| CAN 总线 | CAN | 汽车电子、工业控制（插件扩展） |

## 二、串口通信

### 2.1 参数配置

```json
{
    "type": "serial",
    "port": "COM3",
    "baud_rate": 115200,
    "data_bits": 8,
    "parity": "none",
    "stop_bits": 1,
    "flow_control": "none",
    "timeout_ms": 1000
}
```

#### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|-----|------|-------|------|
| type | string | - | 固定值 `serial` |
| port | string | - | 端口名：Windows `COMx`，Linux `/dev/ttyUSBx` |
| baud_rate | int | 9600 | 波特率 |
| data_bits | int | 8 | 数据位：5, 6, 7, 8 |
| parity | string | none | 校验位：`none`, `odd`, `even`, `mark`, `space` |
| stop_bits | int/string | 1 | 停止位：1, 1.5, 2 |
| flow_control | string | none | 流控制：`none`, `hardware` (RTS/CTS), `software` (XON/XOFF) |
| timeout_ms | int | 1000 | 读超时时间（毫秒） |

#### 支持的波特率

```
标准波特率:
1200, 2400, 4800, 9600, 19200, 38400, 57600, 115200, 230400, 
460800, 921600

高速波特率:
1000000, 1500000, 2000000, 3000000
```

### 2.2 串口设备枚举

**Windows:**
```
COM1, COM2, COM3, ... (系统枚举)
\\.\COM10, \\.\COM11, ... (超过 COM9 需要转义)
```

**Linux:**
```
/dev/ttyS0, /dev/ttyS1, ...     (硬件串口)
/dev/ttyUSB0, /dev/ttyUSB1, ... (USB转串口)
/dev/ttyACM0, /dev/ttyACM1, ... (USB ACM设备)
/dev/ttyAMA0                    (树莓派板载串口)
```

## 三、TCP 客户端

### 3.1 参数配置

```json
{
    "type": "tcp_client",
    "host": "192.168.1.100",
    "port": 5000,
    "timeout_ms": 5000,
    "reconnect": {
        "enabled": true,
        "interval_ms": 3000,
        "max_attempts": 10
    },
    "keepalive": {
        "enabled": true,
        "idle_ms": 60000,
        "interval_ms": 10000,
        "count": 3
    }
}
```

#### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|-----|------|-------|------|
| type | string | - | 固定值 `tcp_client` |
| host | string | - | 服务器地址（IP 或域名） |
| port | int | - | 服务器端口 |
| timeout_ms | int | 5000 | 连接/读写超时 |
| reconnect.enabled | bool | true | 是否自动重连 |
| reconnect.interval_ms | int | 3000 | 重连间隔 |
| reconnect.max_attempts | int | 0 | 最大重连次数（0=无限） |
| keepalive.enabled | bool | true | 是否启用心跳 |
| keepalive.idle_ms | int | 60000 | 空闲时间 |
| keepalive.interval_ms | int | 10000 | 心跳间隔 |
| keepalive.count | int | 3 | 失败次数 |

### 3.2 连接状态

| 状态 | 说明 |
|-----|------|
| `Disconnected` | 未连接 |
| `Connecting` | 连接中 |
| `Connected` | 已连接 |
| `Disconnecting` | 断开中 |
| `Reconnecting` | 重连中 |
| `Error` | 连接错误 |

## 四、TCP 服务器

### 4.1 参数配置

```json
{
    "type": "tcp_server",
    "listen_port": 8080,
    "listen_address": "0.0.0.0",
    "max_clients": 10,
    "timeout_ms": 30000,
    "client_config": {
        "receive_buffer_size": 65536,
        "send_buffer_size": 65536
    }
}
```

#### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|-----|------|-------|------|
| type | string | - | 固定值 `tcp_server` |
| listen_port | int | - | 监听端口 |
| listen_address | string | "0.0.0.0" | 监听地址 |
| max_clients | int | 10 | 最大客户端数 |
| timeout_ms | int | 30000 | 客户端空闲超时 |
| client_config.receive_buffer_size | int | 65536 | 接收缓冲区大小 |
| client_config.send_buffer_size | int | 65536 | 发送缓冲区大小 |

### 4.2 客户端事件

| 事件 | 说明 |
|-----|------|
| `ClientConnected` | 客户端连接 |
| `ClientDisconnected` | 客户端断开 |
| `ClientDataReceived` | 收到客户端数据 |
| `ClientError` | 客户端错误 |

## 五、UDP 通信

### 5.1 参数配置

```json
{
    "type": "udp",
    "mode": "unicast",
    "local_port": 5000,
    "remote_host": "192.168.1.100",
    "remote_port": 5001,
    "multicast_group": "239.0.0.1",
    "buffer_size": 65536
}
```

#### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|-----|------|-------|------|
| type | string | - | 固定值 `udp` |
| mode | string | unicast | 模式：`unicast`, `broadcast`, `multicast` |
| local_port | int | 0 | 本地端口（0=随机） |
| remote_host | string | - | 远程地址 |
| remote_port | int | - | 远程端口 |
| multicast_group | string | - | 组播组地址（multicast 模式） |
| multicast_ttl | int | 1 | 组播 TTL |
| buffer_size | int | 65536 | 缓冲区大小 |

### 5.2 UDP 模式

| 模式 | 说明 |
|-----|------|
| `unicast` | 单播：点对点通信 |
| `broadcast` | 广播：向网络广播数据 |
| `multicast` | 组播：向组播组发送数据 |

## 六、Modbus 协议

### 6.1 Modbus RTU

```json
{
    "type": "modbus_rtu",
    "serial": {
        "port": "COM3",
        "baud_rate": 9600,
        "data_bits": 8,
        "parity": "even",
        "stop_bits": 1
    },
    "slave_id": 1,
    "timeout_ms": 1000,
    "inter_frame_delay_ms": 3.5
}
```

#### Modbus RTU 帧格式

```
┌──────────┬──────────────┬─────────────┬─────────┐
│ 地址(1B) │ 功能码(1B)   │ 数据(N B)   │ CRC(2B) │
└──────────┴──────────────┴─────────────┴─────────┘
```

#### 支持的功能码

| 功能码 | 名称 | 说明 |
|-------|------|------|
| 0x01 | Read Coils | 读取线圈状态 |
| 0x02 | Read Discrete Inputs | 读取离散输入 |
| 0x03 | Read Holding Registers | 读取保持寄存器 |
| 0x04 | Read Input Registers | 读取输入寄存器 |
| 0x05 | Write Single Coil | 写单个线圈 |
| 0x06 | Write Single Register | 写单个寄存器 |
| 0x0F | Write Multiple Coils | 写多个线圈 |
| 0x10 | Write Multiple Registers | 写多个寄存器 |

### 6.2 Modbus TCP

```json
{
    "type": "modbus_tcp",
    "connection": {
        "host": "192.168.1.100",
        "port": 502
    },
    "unit_id": 1,
    "timeout_ms": 5000
}
```

#### Modbus TCP 帧格式

```
┌───────────────────────────────────────────────────┐
│              MBAP Header (7 Bytes)                │
├──────────┬──────────┬──────────┬─────────────────┤
│事务ID(2B)│协议ID(2B)│长度(2B)  │单元ID(1B)       │
└──────────┴──────────┴──────────┴─────────────────┘
├───────────────────────────────────────────────────┤
│              PDU (变长)                           │
├──────────────┬────────────────────────────────────┤
│ 功能码(1B)   │ 数据(N B)                          │
└──────────────┴────────────────────────────────────┘
```

## 七、协议帧格式

### 7.1 通用帧结构

```
┌────────────────────────────────────────────────────────┐
│ 帧头 │ 长度 │ 地址/ID │ 命令 │ 数据 │ 校验 │ 帧尾 │
└────────────────────────────────────────────────────────┘
```

### 7.2 常见帧格式示例

#### 7.2.1 固定长度帧

```
帧头: AA 55
数据: 4 字节
校验: 1 字节 XOR

示例: AA 55 12 34 56 78 49
      │  │  └───┬───┘  │
      │  │      │      └─ 校验 (0x12^0x34^0x56^0x78 = 0x49)
      │  │      └──────── 数据
      └──┴─────────────── 帧头
```

#### 7.2.2 变长帧（长度字段）

```
帧头: AA 55
长度: 2 字节 (大端)，表示后续数据长度
数据: N 字节
校验: 2 字节 CRC16

示例: AA 55 00 05 01 02 03 04 05 12 34
      │  │  └──┬──┘ └────┬────┘ └──┬──┘
      │  │     │        │         └─ CRC16
      │  │     │        └─────────── 数据 (5字节)
      │  │     └──────────────────── 长度 = 5
      └──┴────────────────────────── 帧头
```

#### 7.2.3 分隔符帧

```
起始符: STX (0x02)
结束符: ETX (0x03)
转义符: DLE (0x10)

示例: 02 48 65 6C 6C 6F 03
      │  └─────┬─────┘  │
      │        │        └─ ETX
      │        └────────── 数据 "Hello"
      └─────────────────── STX
```

| 类型 | 大小 | 范围 |

| float64 | 8B | IEEE 754 双精度 |

## 九、心跳与保活

### 9.1 心跳机制

```json
{
    "heartbeat": {
        "enabled": true,
        "interval_ms": 5000,
        "timeout_ms": 15000,
        "max_missed": 3,
        "send_data": "PING",
        "expect_data": "PONG"
    }
}
```

### 9.2 心跳流程

```
┌─────────┐                    ┌─────────┐
│ Client  │                    │ Server  │
└────┬────┘                    └────┬────┘
     │                              │
     │──── PING (每5秒) ────────────▶│
     │                              │
     │◀─────────── PONG ────────────│
     │                              │
     │  (超时15秒未收到响应)         │
     │                              │
     │──── 重连机制触发 ────────────▶│
     │                              │
```

## 十、错误处理

### 10.1 错误码定义

| 错误码 | 名称 | 说明 |
|-------|------|------|
| 0x00 | SUCCESS | 成功 |
| 0x01 | ERR_TIMEOUT | 超时 |
| 0x02 | ERR_CONNECTION_FAILED | 连接失败 |
| 0x03 | ERR_CONNECTION_LOST | 连接丢失 |
| 0x04 | ERR_INVALID_DATA | 无效数据 |
| 0x05 | ERR_CHECKSUM | 校验错误 |
| 0x06 | ERR_BUFFER_OVERFLOW | 缓冲区溢出 |
| 0x07 | ERR_DEVICE_BUSY | 设备忙 |
| 0x08 | ERR_PERMISSION | 权限错误 |
| 0x09 | ERR_NOT_SUPPORTED | 不支持的操作 |
| 0x0A | ERR_INVALID_PARAM | 参数错误 |
| 0xFF | ERR_UNKNOWN | 未知错误 |

### 10.2 重试策略

```json
{
    "retry": {
        "enabled": true,
        "max_attempts": 3,
        "interval_ms": 1000,
        "backoff": "exponential",
        "backoff_factor": 2
    }
}
```

| 策略 | 说明 |
|-----|------|
| `none` | 不重试 |
| `fixed` | 固定间隔重试 |
| `linear` | 线性递增间隔 |
| `exponential` | 指数退避 |

## 十二、附录

### 12.1 端口权限（Linux）

```bash
# 添加用户到 dialout 组（串口权限）
sudo usermod -a -G dialout $USER

# 修改设备权限（临时）
sudo chmod 666 /dev/ttyUSB0

# 创建 udev 规则（永久）
# /etc/udev/rules.d/99-serial.rules
SUBSYSTEM=="tty", ATTRS{idVendor}=="10c4", ATTRS{idProduct}=="ea60", MODE="0666"
```

### 12.2 防火墙配置

**Windows:**
```powershell
# 允许端口入站
netsh advfirewall firewall add rule name="DeviceStudio TCP" dir=in action=allow protocol=tcp localport=8080

# 允许端口出站
netsh advfirewall firewall add rule name="DeviceStudio TCP Out" dir=out action=allow protocol=tcp localport=any
```

**Linux (ufw):**
```bash
# 允许端口
sudo ufw allow 8080/tcp

# 允许组播
sudo ufw allow from 239.0.0.0/8
```
